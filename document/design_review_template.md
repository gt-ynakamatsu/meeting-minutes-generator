# 設計書レビュー記録テンプレート

会議用の議事録フォーマット（アプリ既定のマージ出力）とは別に、**レビュー計画・観点・記録**を人間が手で埋める／音声・テキスト投入前の下書きに使うためのテンプレです。  
アプリで議事録化するときは、会議タイププリセットで **「設計書レビュー」** を選ぶと抽出・統合ヒントが差し込まれます。

## 3. レビュー観点チェックリスト（必要な項目だけ使用）

- [ ] **スコープ・前提** … 対象外・前提条件が文書と一致しているか  
- [ ] **インターフェース** … API・イベント・DB 契約に矛盾・欠落がないか  
- [ ] **非機能** … 性能・可用性・拡張性の記述は妥当か  
- [ ] **セキュリティ・プライバシー** … 認可・秘密情報・ログの扱い  
- [ ] **運用・移行** … デプロイ、ロールバック、監視  
- [ ] **テスト観点** … 受け入れ条件・検証可能な記述  
- [ ] **依存・リスク** … 外部依存、未決事項の明示

## アプリへの入力のヒント

- **音声・動画**: 上記チェックリストを口頭でなぞると、抽出が安定しやすいです。  
- **テキスト**: 設計書の該当章を貼り付けた `.txt` と、別ファイルで「指摘メモ」を **参考資料（メモ）** として渡すと、固有名の照合に使えます。
